# 验收清单(白话版,v0.2 — Ask.app)

每条都写明:**点哪里 → 应该看到什么**。任何一条不对就回去修。

两种验收路径(任选一种,推荐两种都过一遍):

- **.app 路径**:`make build && open dist/Ask.app`(双击启动,菜单栏出现 "Ask")。
- **源码路径**:`source venv/bin/activate && python run.py`,浏览器开 `http://127.0.0.1:8870`。

---

## 1. 首屏不卡白屏

- **看**:页面加载后立刻能看见顶部 logo / 模式 tab / 设置按钮,左侧"会话列表",右侧主区。
- **如果还没配 provider**,主区显示一个大大的"👋 欢迎,先加一个模型再聊吧"和一个"去设置页加第一个模型 →"的按钮 — 不能让你面对空白。

## 2. 加 1 个国内模型,key 故意写错

- **点**:右上角 ⚙️ 设置 → 弹窗里点"+ 添加提供方" → 选一个国内模型(例如 DeepSeek、智谱 GLM、通义千问)。
- **应该看到**:`base_url` 已经预填好,你只需要填显示名 + API Key。
- **故意填一个错的 key**(例如 `sk-bad`),点"🩺 测试"。
- **应该看到**:红色 ✗ 提示 — 例如"鉴权失败(401)"加上 provider 返回的具体报错。错误必须显示在 UI 里,不只是控制台。

## 3. 加 1 个聚合站

- **点**:同样的添加流程,这次选"硅基流动 SiliconFlow"或"OpenRouter"。
- **应该看到**:`base_url` 已经预填好,默认模型列表里已经有 2-3 个流行模型(可改可加)。
- 填上你的真 key,点测试,绿色 ✓"连接 OK"才能保存。

## 4. 单聊模式 — 流式输出

- **点**:顶部 tab 切到"单聊",在模型下拉里选一个你刚配好的。
- **输入**:"用一句话介绍你自己" → 回车。
- **应该看到**:右侧逐字 / 逐 token 出现回答,Markdown 实时渲染(粗体、代码块、列表都能正常显示)。不是"等几秒一次性出现"。
- 同时左侧栏冒出一条新会话,标题自动取你说的前 40 字。

## 5. 对比模式 — 并行两边

- **点**:tab 切到"对比",左右两个下拉各选一个模型(可以是同一个 provider 不同 model)。
- **输入**:同一句话 → 发送。
- **应该看到**:屏幕分成左右两栏,**两边同时**开始流式输出,谁快谁慢一目了然。完事后两条回答都进了 SQLite,刷新页面也还在。

## 6. 辩论模式 — 至少 1 轮主辩+反驳

- **点**:tab 切到"辩论",选 A 方 + B 方,辩论方式选"主辩 + 反方",轮数选 2。
- **输入**:有争议的话题,例如"AI 是否会取代程序员?" → 发送。
- **应该看到**:
  - 先 round 1 — Lead 主辩长篇大论 → Critic 反驳。
  - 再 round 2 — Lead 回应 → Critic 追问。
  - 任意一条助手发言旁有"采纳"按钮,点了变成"已采纳"徽章。

---

## 其他必须点过一遍的

- **左侧栏搜索**:输入会话标题片段,列表实时过滤。
- **删除会话**:鼠标移到会话行,右上的 🗑️ 出现,点了弹"确认删除"对话框,**不会一键删干净**。
- **删除 provider**:同样有二次确认。
- **深色 / 浅色切换**:右上角 ☀️/🌙,切换平滑(不闪白屏)。
- **API Key 输入框**:必须是 `type="password"`,看不到明文;编辑现有 provider 时占位符写"留空保留原有密钥"。
- **关闭弹窗**:`ESC` 键或点击遮罩都能关闭设置弹窗。
- **持久化**:关掉服务再启动,所有会话和历史还在。

## v0.2 新增,要专门点过一遍的

- **讨论模式**:`⌘+4` 或顶 tab "讨论",选两个模型(A 方 / B 方,**没有立场输入框、没有子模式**),最多轮数选 3,问一个有争议的问题(例如"该不该用 SQLite 上 FTS5 还是改用 Meilisearch"),应该看到:
  - 每条消息都按 6 字段【当前判断 / 把握度 / 支撑 / 被对方修正 / 仍坚持 / 需要对方回应】结构输出,不再有 Lead/Critic 的对抗腔调。
  - 双方【把握度】都到 8 以上 + 判断接近时,后端会自动提前停,不一定跑满 3 轮。
  - 最后一条永远由 A 方出一段带绿色边框 + 📌 共识徽章的总结(【共识】/【主要分歧】/【建议给用户的下一步】),旁边有"📋 复制共识"按钮一键拷走。
- **上下文进度条**:多轮聊天后,composer 上方应该出现进度条,数字 = `已用 / 上限`。
  超 70% 变橙色,超 90% 变红色 + 旁边出现 "📦 压缩历史" 按钮 → 点了压缩成一条带 `📦 摘要` 徽章的合成消息。
- **重新生成 / 换模型重生**:最后一条 assistant 气泡右侧应该有 "复制 / ↻ 重新生成 / 换模型重生" 三个按钮,
  点 ↻ 应该把上一轮重新输出;在下拉里换个模型应该用新模型重答(原有的 user 消息不被重复)。
- **⌘+F 全局搜索**:任何时候按 `⌘+F`(.app 模式由菜单栏触发,源码模式由全局快捷键触发)弹出搜索对话框,
  关键词输入后应该跨所有会话搜出来,带高亮 snippet。点结果跳到对应消息。
- **辩论立场**:辩论 tab 下应该有两个"A 方立场 / B 方立场"输入框。填了之后两边模型应该围绕指定立场展开,
  即使逻辑上可以转向也不会轻易倒戈。
- **托盘 / 菜单栏(.app 模式)**:菜单栏应该有 Ask / 文件 / 编辑 / 视图 / 窗口 / 帮助;
  顶部状态栏(右上角)应该有一个 💬 图标,点开显示 / 隐藏窗口 + 新建 + 退出。
  按 `⌘+,` 打开设置,`⌘+N` 新建会话,`⌘+L` 聚焦输入框,`⌘+Q` 退出。
- **About 面板**:菜单栏 `Ask → 关于 Ask`,弹窗显示版本号 0.2.0、Bundle id、数据 / 日志路径。
- **Keychain 模式 (.app)**:在设置里填 API key 保存,然后 `Keychain.app` 里搜 `com.birdindasky.ask` 应该能看到对应条目;
  config.json 里**不应该**出现 `api_key` 字段。

## 自动化层面已覆盖(`pytest -q`,42 用例)

- adapter 注册表、未知 kind 报错、env scrub、CLI 寻 PATH。
- config 默认值 / v1→v2 迁移 / 原子写 / 旧 plain-text key 自动迁入 Keychain。
- sessions / messages CRUD + touch + FTS5 触发器(insert/update/delete 同步)。
- 单聊 / 对比 / 辩论 三种模式的 SSE 事件流(mock provider)。
- HTTP 端到端:provider redact、`/chat` `/compare` `/debate` SSE 格式、`/regenerate`、
  `/summarize`、`/budget`、`/admin/{export,import,wipe}`、`/sessions/search/messages` 中文 trigram 命中、
  并发流 409 守卫、辩论 stance 注入 system prompt、附件文本文件 inline + 图片占位。

如果 `python -m pytest -q` 全绿,说明骨架健康。其余靠这份清单一条条点过去。
